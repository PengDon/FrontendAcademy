# CSS 选择器知识点

## 基础选择器

### 1. 元素选择器
```css
h1 { color: blue; }
```

### 2. ID选择器
```css
#header { background-color: gray; }
```

### 3. 类选择器
```css
.button { padding: 10px; }
```

### 4. 通配符选择器
```css
* { margin: 0; padding: 0; }
```

## 组合选择器

### 1. 后代选择器（空格）
```css
div p { color: green; }
```

### 2. 子选择器（>）
```css
div > p { color: red; }
```

### 3. 相邻兄弟选择器（+）
```css
h2 + p { margin-top: 10px; }
```

### 4. 通用兄弟选择器（~）
```css
h2 ~ p { color: purple; }
```

## 伪类选择器

### 动态伪类
- `:hover`: 鼠标悬停时
- `:active`: 被激活时
- `:focus`: 获得焦点时
- `:link`: 未访问的链接
- `:visited`: 已访问的链接

### 结构伪类
- `:first-child`: 第一个子元素
- `:last-child`: 最后一个子元素
- `:nth-child(n)`: 第n个子元素
- `:nth-of-type(n)`: 同类型中的第n个元素
- `:only-child`: 唯一的子元素
- `:root`: 文档根元素

### 状态伪类
- `:checked`: 选中的表单元素
- `:disabled`: 禁用的表单元素
- `:enabled`: 启用的表单元素

## 属性选择器

```css
/* 包含指定属性 */
[title] { color: blue; }

/* 属性等于指定值 */
[href="https://example.com"] { color: green; }

/* 属性以指定值开头 */
[href^="https"] { color: red; }

/* 属性以指定值结尾 */
[src$=".jpg"] { border: 1px solid black; }

/* 属性包含指定值 */
[class*="btn"] { padding: 5px; }
```

## 选择器优先级

优先级从高到低：

1. **内联样式** (style属性) - 1000
2. **ID选择器** - 0100
3. **类选择器、伪类选择器、属性选择器** - 0010
4. **元素选择器、伪元素选择器** - 0001
5. **通配符选择器** - 0000

### 计算规则
- 优先级相同时，后定义的规则覆盖先定义的规则
- `!important` 可以覆盖任何优先级，但应谨慎使用

## 常见问题与答案

### 1. 为什么我的CSS规则没有生效？
**答案：** 可能的原因：
- 选择器优先级不够高
- 选择器写错了
- CSS属性拼写错误
- 元素样式被更具体的选择器覆盖
- 存在`!important`标记的其他规则

### 2. 什么是CSS特异性？
**答案：** CSS特异性是浏览器确定哪个CSS规则应用于元素的机制，基于选择器的类型和组合。它决定了在多个规则针对同一元素时，哪个规则具有更高的优先级。

### 3. 什么时候应该使用ID选择器，什么时候应该使用类选择器？
**答案：** 
- ID选择器用于唯一标识一个元素，整个页面中只能出现一次
- 类选择器用于标识一组具有相同样式的元素，可以多次使用
- 建议优先使用类选择器，因为它们的优先级较低，更容易被覆盖和维护

### 4. 如何选择第n个元素？
**答案：** 使用`:nth-child()`或`:nth-of-type()`伪类。例如：
- `:nth-child(3)` 选择第3个子元素
- `:nth-child(2n)` 选择所有偶数位置的子元素
- `:nth-of-type(3)` 选择同类型中的第3个元素

## 最佳实践

1. **避免使用ID选择器**：除非必要，优先使用类选择器
2. **避免使用`!important`**：尽量通过调整选择器优先级来解决冲突
3. **使用BEM命名规范**：帮助组织和维护CSS代码
4. **保持选择器简洁**：避免过度嵌套，通常不超过3层
5. **使用CSS变量**：便于主题切换和维护
6. **为选择器添加注释**：解释复杂选择器的用途
